# GitHub同步工具解决方案

## 问题原因
原脚本 `sync_to_github.bat` 可能因为编码问题（UTF-8 BOM）或复杂的命令结构导致在某些Windows系统上无法正常运行。

## 解决方案
我们提供了一个使用纯ASCII编码的简化版同步脚本：`ascii_sync.bat`

## 使用方法
1. 确保已安装Git（下载地址：https://git-scm.com/downloads）
2. 找到项目文件夹中的 `ascii_sync.bat` 文件
3. 直接双击该文件运行
4. 按照屏幕提示操作

## 脚本功能
该脚本会自动执行以下操作：
1. 检查Git是否已安装
2. 添加所有新增和修改的文件
3. 提交更改（使用时间戳作为提交信息）
4. 推送到GitHub仓库

## 注意事项
1. 首次运行时可能需要输入GitHub账号密码
2. 确保网络连接正常
3. 如果遇到权限问题，请检查您的GitHub仓库访问权限

## 紧急备选方案
如果双击脚本仍然没有反应，可以尝试以下方法：

1. 打开命令提示符（CMD）
2. 导航到项目文件夹（使用 `cd` 命令）
3. 输入以下命令并按Enter：
   ```
   ascii_sync.bat
   ```

或者直接在命令行中运行以下Git命令：
```
git add .
git commit -m "Auto sync"
git push https://github.com/waynezyx007/student-picker-app.git master
```

## 技术支持
如果仍然遇到问题，请检查：
- Git是否正确安装（在命令行输入 `git --version` 测试）
- 是否有网络连接到GitHub
- 是否有仓库访问权限

祝您使用愉快！
